# 需求分析报告

## 程序
- 程序名: ORDPRC
- 目标: 自动处理待处理订单并记录处理结果

## 原始需求摘录
程序名：ORDPRC
功能：订单处理程序

主要功能：
1. 读取 ORDPF 订单文件中的待处理订单（状态 = 01）
2. 验证客户信息（查询 CUSTMF 客户主文件）
3. 验证库存（查询 INVPF 库存文件，检查数量是否充足）
4. 更新订单状态：
   - 通过：将 ORDPF 状态更新为 02（已确认）
   - 库存不足：将 ORDPF 状态更新为 09（库存不足）
   - 客户无效：将 ORDPF 状态更新为 08（客户无效）
5. 记录处理日志到 ORDLOGPF

涉及文件：
- ORDPF：订单文件（PF），关键字段：订单号(ORDNO)
- CUSTMF：客户主文件（PF），关键字段：客户号(CUSTNO)
- INVPF：库存文件（PF），关键字段：物料号(ITEMNO)
- ORDLOGPF：订单日志文件（PF），记录处理结果

性能要求：处理 1000 条订单 < 30 秒

## 处理步骤
- 1. 读取 ORDPF 订单文件中的待处理订单（状态 = 01）
- 2. 验证客户信息（查询 CUSTMF 客户主文件）
- 3. 验证库存（查询 INVPF 库存文件，检查数量是否充足）
- 4. 更新订单状态：
- 5. 记录处理日志到 ORDLOGPF

## 关键文件
- ORDPF: 订单主文件
- CUSTMF: 客户主文件
- INVPF: 库存文件
- ORDLOGPF: 处理日志文件

## 业务规则
- 仅处理状态 = 01 的订单
- 客户不存在或无效时状态更新为 08
- 库存不足时状态更新为 09
- 所有通过验证的订单状态更新为 02
- 每笔订单处理结果必须写入日志

## 风险与待澄清
- ORDLOGPF 的字段结构需要在落地时与现有 DDS 定义核对。
