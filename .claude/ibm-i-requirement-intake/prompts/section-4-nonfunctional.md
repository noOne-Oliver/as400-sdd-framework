# AS400 需求录入 - 第 4 章：非功能需求

## 目的

明确系统的性能、可用性、安全和兼容性要求，作为技术设计和验收的依据。

## 引导问题

### 4.1 性能需求

- 单笔交易响应时间要求（秒）
- 批量处理吞吐量（笔/分钟）
- 并发用户数
- 报表生成时间

### 4.2 可用性需求

- 营业时间要求（7x24 / 5x8 / 批量窗口）
- 故障恢复时间目标（RTO）
- 数据恢复点目标（RPO）

### 4.3 安全需求

- 权限级别划分（操作员/主管/系统）
- 敏感数据处理（密码/金额/个人信息）
- 审计日志要求

### 4.4 兼容性需求

- IBM i 系统版本（7.3 / 7.4 / 7.5）
- 是否需与 DMS / FTP / API 集成
- RPGLE 版本（RPG / RPGLE / SQLRPGLE）

## 输出要求

在 `requirements/{project-name}-requirement.md` 中填充第 4 章内容。
