# AS400 需求录入（Requirement Intake）

## Overview

本技能引导用户逐步完成 IBM i (AS/400) 项目的需求录入，基于 build-agent-skill Document Chain 设计。输出标准 9 章节需求文档，供后续 Requirement Normalizer、Functional Spec、Technical Design 使用。

## 9 章节结构

| # | 章节 | 填写角色 | 输出章节锚点 |
|---|------|----------|-------------|
| 1 | 背景与动机 | 业务/BA | #1-背景与动机 |
| 2 | 涉众分析 | 业务/BA | #2-涉众分析 |
| 3 | 功能需求 | 业务/BA | #3-功能需求 |
| 4 | 非功能需求 | 技术/架构师 | #4-非功能需求 |
| 5 | 接口需求 | 技术/架构师 | #5-接口需求 |
| 6 | 影响范围分析 | 技术/架构师 | #6-影响范围分析 |
| 7 | 约束条件 | 技术/架构师 | #7-约束条件 |
| 8 | 附录 | 业务/BA | #8-附录 |
| 9 | 填写说明 | 自动汇总 | #9-填写说明 |

## 工作流程

```
用户触发 → 确认项目名称 → 逐章节引导 → 自动填充元信息 → 输出最终文档
```

### 步骤 1: 确认项目信息

在开始前，确认：
- **项目名称**（用于输出文件名）
- **录入编号**（格式：RN-YYYYMMDD-NNN）
- **录入人**
- **业务领域**

### 步骤 2: 逐章节引导

按顺序加载各章节引导 prompt（见 `prompts/` 目录）：
- 每次只加载一个章节，待用户完成后加载下一个
- 自动在文档中填充章节锚点和内容

### 步骤 3: 输出最终文档

所有章节完成后，汇总输出：

```markdown
# AS400 需求录入文档
> 版本：V1.0 | 状态：草稿 | 录入编号：{录入编号}

[所有章节内容...]
```

**输出路径**: `requirements/{project-name}-requirement.md`

## 章节编号规则

| 类型 | 编号格式 | 示例 |
|------|----------|------|
| 用例 | UC-01, UC-02... | UC-01 |
| 业务规则 | BR-01, BR-02... | BR-01 |
| 录入编号 | RN-YYYYMMDD-NNN | RN-20250416-001 |

## 后续流程

需求录入完成后，文档进入后续 Document Chain：

```
Requirement Intake → Requirement Normalizer → Functional Spec → Technical Design → Program Spec → Code
```

## 使用示例

### 触发方式

```
请帮我录入一个 AS400 需求
```

### 对话示例

```
用户：订单处理模块需要增加库存扣减功能
助手：好的，我来帮你录入这个需求。请提供以下信息：
      - 项目名称：订单库存联动
      - 录入编号：RN-20250416-001
      - 录入人：[你的名字]
      确认后我们从第一章开始。
```

## 注意事项

1. **章节顺序**：必须按顺序完成，前一章节是后续章节的上下文
2. **编号管理**：UC 和 BR 编号全局连续，不重复
3. **元信息**：录入编号、日期、录入人在 init 阶段确认
4. **输出路径**：统一输出到 `requirements/` 目录
